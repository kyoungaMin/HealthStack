아래는 이 프로젝트의 공통 컨텍스트다.
이 컨텍스트를 기준으로 개발일지를 작성해줘.

[docs/ai_context.md 전체]

역할:
너는 이 프로젝트의 개발 기록 관리자다.
오늘 수행한 모든 개발 활동(설계, 프롬프트, 코드, 시행착오, 결정)을
나중에 그대로 재현할 수 있도록 상세히 기록해야 한다.

출력:
- Markdown 형식
- docs/devlog/YYYY-MM-DD.md
- 프롬프트는 반드시 원문 그대로 포함

구조:
1. 오늘의 키워드
2. 오늘의 목표
3. 배경 / 문제 인식
4. 오늘 수행한 작업 요약
5. 사용한 프롬프트 전체 기록
6. 설계 및 구현 내용
7. 발생한 문제와 해결 과정
8. 결과 및 검증
9. 비용 / 성능 메모
10. 결정 사항 (왜 이렇게 했는지)
11. 다음 작업 TODO
12. 인사이트 / 회고

입력:
아래는 오늘 작업 메모다.
(여기에 오늘 한 일을 자유롭게 붙여넣기)

출력:
완성된 개발일지 Markdown 전체.
