# 개발일지 (Development Log)

## 개요
일자별 개발 작업내역을 기록하는 폴더입니다.

## 파일 명명 규칙
- 파일명: `YYYY-MM-DD.md` (예: `2026-01-23.md`)
- 하루 1개 문서 원칙

## 작성 원칙
1. **사실 기반** - 객관적이고 간결하게 작성
2. **투명성** - 이슈/오류는 숨기지 않고 원인·조치·재발방지까지 기록
3. **추적 가능** - 프롬프트는 실제 입력한 문장 그대로 기록
4. **연결성** - 관련 이슈/PR/문서 링크 첨부

## 템플릿 구조

```markdown
# YYYY-MM-DD

## 요약
- 오늘의 핵심 결과 1~3줄

## 프롬프트
- [요청] 실제 입력한 프롬프트/지시사항

## 작업내역
- [기능] 구현한 기능 설명
- [설정] 환경 설정/연동 작업
- [수정] 버그 수정/리팩토링

## 적용사항
- [파일] 생성/수정된 파일 목록
- [패키지] 추가된 의존성
- [설정] 환경 변수/설정 변경

## 오류
- [증상] 어떤 오류가 발생했는가
- [원인] 왜 발생했는가
- [영향] 어떤 영향이 있었는가

## 해결
- [조치] 어떻게 해결했는가
- [근거] 왜 이 방법을 선택했는가
- [결과] 해결 여부 및 검증 방법

## 결과물
- [산출물] 완성된 파일/기능/배포 정보
- [링크] 관련 PR/이슈/문서

## 회고
- [좋았던 점] 
- [아쉬운 점]

## 다음 행동
- [To-Do] 후속 작업/미해결 이슈
```
