# Health Stack 통합 기획안

### ― 증상·처방전 기반 개인 맞춤 건강 이해 & 생활 추천 서비스 ―

---

## 1. 서비스 한 문장 정의

> Health Stack은 사용자가 입력한 증상과 처방전을 바탕으로,
약·건강정보·동의보감·음식·지역 식당·판매처를 하나의 맥락으로 연결해
“내 몸에 지금 필요한 선택지”를 설명해주는 서비스이다.
> 
- 치료 ❌
- 처방 변경 ❌
- 과장 ❌
- **이해 + 판단 보조 + 생활 선택 가이드 ⭕**

---

## 2. MVP 범위 정의

> **MVP 목적**: 복잡한 건강 정보를 **증상·처방전 입력만으로 이해 가능한 언어로 설명**하고,
> **바로 실천 가능한 음식 선택**까지 연결되는지 검증

### 2.1 MVP 핵심 기능 (Core)

| # | 기능 | 설명 | 심사 포인트 |
|---|------|------|------------|
| ① | **입력 방식** | 증상 텍스트 / 처방전 사진 / 동시 입력 | 실제 사용자 흐름 명확 |
| ② | **처방전 → 약 추출** | OCR로 약 이름 인식 → 카테고리 수준 | 기술적 도전 + 현실적 범위 |
| ③ | **몸 상태 요약** | 생활·식이 관점 요약 (의학 진단 ❌) | 의료 규제 회피 + 공감 |
| ④ | **동의보감 음식 추천** | 3개 고정 + 현대어 요약 | **핵심 차별화** |
| ⑤ | **근거 연결** | PubMed 논문 1~2개 링크 + 요약 | 민간요법 아님 증명 |
| ⑥ | **실행 연결** | 유튜브 영상 1개 + 한 줄 팁 | 바로 실천 가능 |

### 2.2 MVP 제외 기능 (Backlog)

- 지역 음식점 지도 추천
- 판매 사이트 링크 연결
- 사용자 데이터 누적 관리
- 건강기능식품(건기식) 연결

## 3. 사용자 입력 방식 (3가지 케이스)

Health Stack은 **입력 방식이 달라도 동일한 결과 구조**로 수렴한다.

### 3.1 입력 유형 A : 증상만 입력

**예시**

- “속이 더부룩해요”
- “잠을 잘 못 자요”
- “손발이 차요”

**의미**

- 아직 병원 진단·처방이 없는 상태
- 사용자는 “생활 관리 관점”의 정보를 원함

---

### 3.2 입력 유형 B : 처방전만 입력

**예시**

- 처방전 사진 업로드
- 약 이름 직접 입력

**의미**

- 이미 치료 중
- 사용자는 “지금 먹는 약을 이해하고 싶음”

---

### 3.3 입력 유형 C : 증상 + 처방전 함께 입력 (핵심)

**예시**

- “혈압약을 먹고 있는데 자꾸 어지러워요”
- “위장약 먹는 중인데 소화에 좋은 음식은?”

**의미**

- Health Stack의 **가장 강력한 시나리오**
- 약 정보 + 생활 선택을 함께 제시 가능

---

## 4. 공통 처리 구조 (입력 이후 내부 흐름)

입력 방식이 달라도 내부 처리 구조는 동일하다.

```
[사용자 입력]
   ↓
[입력 해석]
- 증상 분류
- 처방약 인식
   ↓
[Health Stack 구성]
- 현재 복용 약
- 증상 맥락
- 주의 조건
   ↓
[정보 레이어 결합]
- 현대의학 정보
- 동의보감 생활 가이드
   ↓
[결과 제공]
- 설명
- 추천
- 선택지

```

## 5. 제공 정보 구성 (사용자가 실제로 받는 것)

### 5.1 기본 설명 영역 (모든 케이스 공통)

### ① 지금 상태에 대한 설명

- 증상 또는 처방약이 **무엇을 의미하는지**
- 왜 이런 증상이 생길 수 있는지 (단정 ❌)

### ② 주의사항 요약

- 꼭 조심해야 할 경우
- 약·건기식·생활 습관 관련 주의 문구

> ✔ 전문용어 최소화
> 
> 
> ✔ “~일 수 있습니다” 톤 유지
> 

---

## 6. 동의보감 기반 증상 → 식재료 추천 (핵심 차별점)

### 6.1 역할 정의

- 치료 ❌
- 약 대체 ❌
- **생활 속에서 선택 가능한 음식 방향 제시 ⭕**

### 6.2 제공 내용

- 증상에 따라 도움이 될 수 있는 **식재료 목록**
- 왜 이 식재료가 권장되는지 (동의보감 근거)
- 피하면 좋은 음식 방향

**예시**

- 소화불량 → 생강, 대추
- 냉증 → 계피, 부추

---

## 7. 식재료 확장 콘텐츠 (사용자 체감 가치)

### 7.1 요리·활용 동영상

- 식재료별 YouTube 영상
- 요리법 / 차 / 일상 섭취 방법
- “이렇게 활용할 수 있어요” 중심

---

### 7.2 지역 음식점 추천 ⭐

- 사용자의 현재 위치 기준
- 추천 식재료와 **연결된 메뉴를 제공하는 식당**
- 카카오/네이버 지도 API 기반

**제공 정보**

- 식당명
- 카테고리
- 거리
- 평점
- 대표 메뉴 (식재료 근거 표시)

> ✔ “왜 이 식당이 추천됐는지”를 항상 설명
> 

---

### 7.3 판매 사이트 정보 제공

- 해당 식재료를 **직접 구매할 수 있는 링크**
- 재료 단위 제공 (브랜드 비교 ❌)
- 신뢰도 레벨 표시 (공식/일반)

---

## 8. 결과 화면의 최종 구조 (사용자 관점)

```
1. 지금 상태 요약
2. 약 / 증상 이해 설명
3. 주의사항
4. 동의보감 기반 식재료 추천
5. 활용 방법 (영상)
6. 주변 음식점 추천
7. 재료 구매 링크

```

👉 사용자는 **“정보 → 선택 → 행동”**까지 한 화면에서 이어진다.
