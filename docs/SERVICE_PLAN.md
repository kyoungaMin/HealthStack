## 1. 기획 배경

### 1.1 현대인의 건강 환경

- 성인 다수는 **성인병·만성질환** 보유
- **처방약을 장기·복수 복용**
- 동시에 **건강기능식품을 일상적으로 섭취**
- 음식·식단 정보는 유튜브/블로그 등 **검증되지 않은 채 범람**

문제는:

- 약 정보는 병원
- 건기식 정보는 광고
- 음식 정보는 콘텐츠
    
    👉 **모두 분리되어 제공됨**
    

---

### 1.2 사용자의 실제 니즈

현대인이 진짜로 알고 싶은 질문:

1. 이 **처방약은 무슨 역할**을 하나?
2. 꼭 알아야 할 **부작용과 주의점**은?
3. **건강기능식품과 같이 먹어도 되는가?**
4. 같이 먹는다면 **복용 시간은 나눠야 하나?**
5. 약 말고 **일상에서 내가 선택할 수 있는 관리 방법**은?
6. 이 모든 것을 **잊지 않게 관리해줄 수는 없나?**

## 4. 정보 레이어 설계

### 4.1 현대의학 레이어 (검증 정보)

- 약의 효능·부작용·경고
- 약 × 약 / 약 × 건강기능식품 상호작용
- 근거 수준 표시
    - 확실
    - 가능성 있음
    - 근거 부족

> 이미 복용 중인 약을 “이해”하게 하는 영역
>

- **증상에 따른 음식·식단·식재료 방향성 제공**

**역할**

> 약 외에 내가 선택할 수 있는
> 
>

>

## 5. 핵심 기능 설계

### 5.1 처방약 정보 요약

- 왜 쓰는 약인지 (효능)
- 꼭 알아야 할 부작용
- 주의 대상 (고령, 임신, 간·신장 질환 등)

✔ 전문용어 최소화

✔ 판단에 필요한 정보만 제공

---

### 5.2 약 × 건강기능식품 조합 분석 (킬러 기능)

- 흡수 방해
- 효과 증감
- 부작용 위험

**표현 방식**

- 🟢 보고된 문제 없음
- 🟡 가능성 있음
- 🔴 주의 필요 (의료진 상담 권장)

⚠️ “먹어도 된다 / 안 된다” 단정 ❌

---

### 5.3 복용 시간표 + 알림 (개인화 핵심 기능)

### 기능 개요

- 공복 / 식후 / 취침 전 조건 설정
- 간격 필요 항목 자동 분리
- 하루 복용 시간표 자동 생성
- 푸시 알림 제공

### 알림 톤 예시

- “지금은 혈압약(식후) 복용 시간이에요.”
- “마그네슘은 철분과 2시간 간격을 권장해요.”

👉 **정보보다 ‘기억을 대신해주는 기능’**

---

### 5.4 증상 기반 건강식단 추천 ⭐

### 원칙

- 치료 식단 ❌
- 증상 관리 관점 ⭕
- 자극 최소화
- 약·건기식과 충돌 없는 구성

### 제공 내용

- 오늘의 식단 방향 설명
- 아침 / 점심 / 저녁 식단
- 오늘 피하면 좋은 음식
- 복용 중인 약 고려 문구

---

### 5.5 동의보감 기반 콘텐츠 확장

- 증상 → 식재료 매핑
- 요리법 동영상
- 추천 레시피
- 식재료 판매 링크
    
    (재료 단위, 비교·추천 없음)

## 6. 사용자 가입 · 계정 관리 설계

### 6.1 가입 원칙

- 가입 장벽 최소화
- 민감 개인정보 수집 ❌
- 소셜 로그인 중심

### 6.2 지원 로그인

- Google 계정 로그인
- Kakao 계정 로그인

✔ 이메일·비밀번호 가입 없음 (초기)

✔ 기능 체험 후 저장/알림 시 로그인 유도

---

### 6.3 사용자 정보 최소화

**필수**

- user_id
- 로그인 제공자(Google/Kakao)
- 생성일

**선택**

- 이메일(영수증/결제용)

❌ 수집하지 않음

- 실명, 주민번호, 주소, 전화번호, 성별, 나이

---

### 6.4 데이터 권리

- 언제든 데이터 삭제
- 계정 탈퇴 시 개인 데이터 즉시 파기
- 리포트는 생성 시점 스냅샷으로 보관
