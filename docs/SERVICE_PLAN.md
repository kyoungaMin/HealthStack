# Health Stack 통합 기획안

### ― 증상·처방전 기반 개인 맞춤 건강 이해 & 생활 추천 서비스 ―

---

## 1. 서비스 한 문장 정의

> Health Stack은 사용자가 입력한 증상과 처방전을 바탕으로,
약·건강정보·동의보감·음식·지역 식당·판매처를 하나의 맥락으로 연결해
“내 몸에 지금 필요한 선택지”를 설명해주는 서비스이다.
> 
- 치료 ❌
- 처방 변경 ❌
- 과장 ❌
- **이해 + 판단 보조 + 생활 선택 가이드 ⭕**

---

## 2. 사용자 입력 방식 (3가지 케이스)

Health Stack은 **입력 방식이 달라도 동일한 결과 구조**로 수렴한다.

### 2.1 입력 유형 A : 증상만 입력

**예시**

- “속이 더부룩해요”
- “잠을 잘 못 자요”
- “손발이 차요”

**의미**

- 아직 병원 진단·처방이 없는 상태
- 사용자는 “생활 관리 관점”의 정보를 원함

---

### 2.2 입력 유형 B : 처방전만 입력

**예시**

- 처방전 사진 업로드
- 약 이름 직접 입력

**의미**

- 이미 치료 중
- 사용자는 “지금 먹는 약을 이해하고 싶음”

---

### 2.3 입력 유형 C : 증상 + 처방전 함께 입력 (핵심)

**예시**

- “혈압약을 먹고 있는데 자꾸 어지러워요”
- “위장약 먹는 중인데 소화에 좋은 음식은?”

**의미**

- Health Stack의 **가장 강력한 시나리오**
- 약 정보 + 생활 선택을 함께 제시 가능

---

## 3. 공통 처리 구조 (입력 이후 내부 흐름)

입력 방식이 달라도 내부 처리 구조는 동일하다.

```
[사용자 입력]
   ↓
[입력 해석]
- 증상 분류
- 처방약 인식
   ↓
[Health Stack 구성]
- 현재 복용 약
- 증상 맥락
- 주의 조건
   ↓
[정보 레이어 결합]
- 현대의학 정보
- 동의보감 생활 가이드
   ↓
[결과 제공]
- 설명
- 추천
- 선택지

```

---

## 4. 제공 정보 구성 (사용자가 실제로 받는 것)

### 4.1 기본 설명 영역 (모든 케이스 공통)

### ① 지금 상태에 대한 설명

- 증상 또는 처방약이 **무엇을 의미하는지**
- 왜 이런 증상이 생길 수 있는지 (단정 ❌)

### ② 주의사항 요약

- 꼭 조심해야 할 경우
- 약·건기식·생활 습관 관련 주의 문구

> ✔ 전문용어 최소화
> 
> 
> ✔ “~일 수 있습니다” 톤 유지
> 

---

## 5. 동의보감 기반 증상 → 식재료 추천 (핵심 차별점)

### 5.1 역할 정의

- 치료 ❌
- 약 대체 ❌
- **생활 속에서 선택 가능한 음식 방향 제시 ⭕**

### 5.2 제공 내용

- 증상에 따라 도움이 될 수 있는 **식재료 목록**
- 왜 이 식재료가 권장되는지 (동의보감 근거)
- 피하면 좋은 음식 방향

**예시**

- 소화불량 → 생강, 대추
- 냉증 → 계피, 부추

---

## 6. 식재료 확장 콘텐츠 (사용자 체감 가치)

### 6.1 요리·활용 동영상

- 식재료별 YouTube 영상
- 요리법 / 차 / 일상 섭취 방법
- “이렇게 활용할 수 있어요” 중심

---

### 6.2 지역 음식점 추천 ⭐

- 사용자의 현재 위치 기준
- 추천 식재료와 **연결된 메뉴를 제공하는 식당**
- 카카오/네이버 지도 API 기반

**제공 정보**

- 식당명
- 카테고리
- 거리
- 평점
- 대표 메뉴 (식재료 근거 표시)

> ✔ “왜 이 식당이 추천됐는지”를 항상 설명
> 

---

### 6.3 판매 사이트 정보 제공

- 해당 식재료를 **직접 구매할 수 있는 링크**
- 재료 단위 제공 (브랜드 비교 ❌)
- 신뢰도 레벨 표시 (공식/일반)

---

## 7. 결과 화면의 최종 구조 (사용자 관점)

```
1. 지금 상태 요약
2. 약 / 증상 이해 설명
3. 주의사항
4. 동의보감 기반 식재료 추천
5. 활용 방법 (영상)
6. 주변 음식점 추천
7. 재료 구매 링크

```

👉 사용자는 **“정보 → 선택 → 행동”**까지 한 화면에서 이어진다.

---

## 8. Health Stack의 본질적 차별점

| 기존 서비스 | Health Stack |
| --- | --- |
| 정보 나열 | 맥락 기반 설명 |
| 약 / 음식 분리 | 하나의 스택으로 통합 |
| 광고 중심 | 근거·출처 중심 |
| 단기 조회 | 누적 관리 |

---

## 9. 이 기획안의 핵심 요약

> Health Stack은
> 
> 
> “무엇을 먹어야 하나”를 말하지 않는다.
> 
> 대신
> 
> **지금 내 몸 상태에서 어떤 선택지가 있는지 이해하게 해준다.**
