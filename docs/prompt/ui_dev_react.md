너는 frontend-tailwind-engineer agent다.
목표는 “밥이약이다” 동의보감 기반 건강 음식 추천 서비스의
React + TailwindCSS UI/UX 프로토타입을 만드는 것이다.

❗중요 UX 원칙
- 탭 UI는 사용하지 않는다
- 한 화면 안에서 “자연스러운 스크롤 흐름”으로 정보를 소비하게 만든다
- 사용자는 ‘검색 → 이해 → 신뢰 → 행동(영상 시청 / 구매)’ 순서로 움직인다
- 의료 서비스가 아닌 “생활 건강 가이드” 톤을 유지한다

---

## 🎯 핵심 사용자 시나리오
1. 사용자가 증상을 입력한다 (예: 불면, 피로, 소화불량)
2. 시스템은 해당 증상에 도움 될 수 있는 음식/재료를 기반으로
   - 관련 동영상 3개
   - 주요 재료 효능/부작용
   - 구매 가능한 판매처
   를 **한 페이지에 섹션으로 나열**한다

---

## 🧱 전체 레이아웃 구조 (Single Page, Section 기반)

### 0️⃣ App Shell
- 상단 고정 Header
  - 로고 + 서비스명 “밥이약이다”
  - 서브 카피: “약보다 음식, 치료보다 예방”
- 최대 폭: max-w-6xl / 가운데 정렬
- 배경: 밝은 화이트 / 섹션 단위로 옅은 구분

---

### 1️⃣ 증상 입력 섹션 (Hero + Search)
**페이지 최상단**

- 타이틀:
  “지금 어떤 증상이 불편하신가요?”
- 입력창:
  - placeholder: “예: 잠이 잘 안 와요, 피로가 계속돼요”
- 검색 버튼 (Primary, red accent)
- 검색 시 아래 섹션들이 순차적으로 나타나는 UX (조건부 렌더링)

---

### 2️⃣ 증상에 따른 추천 음식 요약 섹션
**검색 직후 노출**

- 섹션 타이틀:
  “이 증상에 도움이 될 수 있는 음식”
- 카드 2~3개 (가로 스크롤 or grid)
  - 음식명
  - 한 줄 현대어 요약
    (예: “대추는 긴장을 완화하고 숙면에 도움을 줄 수 있어요”)
  - “자세히 보기” → 아래 섹션으로 스크롤

---

### 3️⃣ 🎥 추천 동영상 섹션 (핵심)
**가장 시각적으로 강조**

- 섹션 타이틀:
  “이런 영상이 도움이 될 수 있어요”
- 유튜브 스타일 카드 3개 (고정 3)
  - 썸네일
  - 제목
  - 채널명
  - 재생 시간
- 클릭 시:
  - 새 탭으로 유튜브 이동
  - (placeholder로 처리)

UX 포인트:
- “의사가 말해주는” 느낌 ❌
- “설명해주는 영상” 느낌 ⭕
- 카드 크기 큼 / 신뢰감 있는 여백

---

### 4️⃣ 🌿 주요 재료 효능 & 부작용 섹션 (정보 신뢰 구간)
**텍스트 중심, 차분한 톤**

- 섹션 타이틀:
  “이 음식에 사용되는 주요 재료”
- 재료 리스트 (Vertical list)

각 재료 카드:
- 재료명 (ex. 인삼)
- 효능 (현대어 요약, 2~3줄)
- 부작용/주의사항 (회색/아이콘으로 구분)
  - “과다 섭취 시…”
  - “고혈압/임산부 주의” 등
- “동의보감 원문 보기” 토글 (접힘 기본)

---

### 5️⃣ 🛒 판매 사이트 섹션 (행동 유도)
**가장 하단, 가볍게**

- 섹션 타이틀:
  “이 재료를 이렇게 구매할 수 있어요”
- 리스트 형태 (카드 X)
  - 제품명
  - 형태 (차 / 환 / 원물)
  - 가격대
  - “구매하러 가기” 버튼
- 버튼은 외부 링크 아이콘 포함
- 과한 CTA 금지 (광고 느낌 ❌)
