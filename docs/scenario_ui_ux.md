# 화면 시나리오 (UI/UX) - 증상 검색 및 처방전 검색

## 1. 개요
사용자가 '증상 입력' 또는 '처방전 업로드'를 통해 건강 정보를 얻는 과정을 단계별로 설계합니다.
**핵심 가치:** 불안감 해소, 쉽고 친근한 안내, 정확한 정보 확인 (Human-in-the-loop).

---

## 2. 시나리오 A: 증상으로 검색 (Symptom Search)

### [Step 1] 메인/검색 화면
**화면 구성:**
- **타이틀:** "어디가 불편하신가요?" (따뜻한 어조)
- **검색창:** "증상을 입력해주세요 (예: 소화불량, 두통, 감기기운)"
- **추천 태그:** [ #체했을때 ] [ #머리아픔 ] [ #기침 ] (자주 찾는 증상)

**사용자 액션:**
- 검색창에 "머리가 지끈거리고 소화가 안돼" 입력 후 돋보기 클릭.

### [Step 2] 증상 분석 및 확인 (Interim Check)
**화면 구성:**
- **상태 메시지:** "잠시만요! 🧐 입력하신 내용을 분석하고 있어요..." (애니메이션 1-2초)
- **증상 키워드 확인:**
  > "제가 이해한 증상이 맞나요?"
  > [ #머리지끈거림 (x) ] [ #소화불량 (x) ]  [ + 증상 추가 ]
- **관련 질문 (옵션):**
  > "혹시 이런 증상도 있으신가요?"
  > [ ] 속이 메스꺼움  [ ] 어지러움  [ ] 열이 남

**사용자 액션:**
- 키워드가 맞으면 "네, 맞아요! 결과 보기 👉" 버튼 클릭.

### [Step 3] 검색 결과 및 선택 (Selection)
**화면 구성:**
- **타이틀:** "비슷한 증상을 찾았어요. 본인의 상황과 가장 비슷한 것을 선택해주세요."
- **리스트 (카드 형태):**
  1. **[동의보감] 두풍증 (Head Wind)**
     - "머리가 지끈거리고, 바람을 쐬면 더 아픈 증상입니다."
  2. **[동의보감] 식적 (Food Stagnation)**
     - "체한 것처럼 명치가 답답하고 머리가 같이 아픈 증상입니다."
  3. **[현대 의학] 긴장성 두통**
     - "스트레스나 피로로 인해 머리가 조이는 듯한 통증입니다."

**사용자 액션:**
- "식적" 카드 선택 후 "상세 분석 보기" 클릭.

### [Step 4] 최종 리포트 (Final Report)
**화면 구성:**
- **진단 요약:** "**식적(체기)**으로 인한 두통일 가능성이 높습니다."
- **맞춤 추천 (Food Therapy):**
  > "동의보감에서는 **'무(Radish)'**가 소화를 돕고 두통을 줄인다고 해요."
  > "오늘 저녁 메뉴로 **시원한 무국**이나 **생강차** 어떠세요? 🥣🍵"
- **현대적 조언:** "소화제를 드신다면 **'베아제'** 같은 소화효소제가 도움이 될 수 있습니다."
- **주의사항:** "증상이 심해지면 꼭 병원을 방문하세요."

## 3. 시나리오 B: 처방전으로 검색 (Prescription OCR)

### [Step 1] 메인/업로드 화면
**화면 구성:**
- **타이틀:** "처방전을 찍어주세요 📸"
- **업로드 영역:** [ 카메라 아이콘 ] "약 봉투나 처방전을 찍으면, 어떤 약인지 알려드릴게요."
- **가이드:** "글자가 잘 보이게 밝은 곳에서 찍어주세요."

**사용자 액션:**
- 처방전 사진 촬영 또는 앨범에서 선택 업로드.

### [Step 2] 스캔 및 인식 확인 (OCR Check)
**화면 구성:**
- **상태 메시지:** "열심히 글자를 읽고 있어요... 🧐" (스캔 애니메이션)
- **인식 결과 확인:**
  > "약 이름을 이렇게 읽었어요. 맞나요?"
  > 💊 **타이레놀이알서방정** [수정]
  > 💊 **소론도정** [수정]
  > 💊 **스티렌투엑스정** [수정]
- **인식 실패 시:** "잘 안 보이는 글자가 있어요. 직접 입력해주시겠어요?"

**사용자 액션:**
- 약 이름 확인 및 오타 수정 후 "분석하기" 클릭.

### [Step 3] 약물 정보 매칭 및 선택 (Drug Matching)
**화면 구성:**
- **타이틀:** "찾으시는 약이 이 약이 맞나요?"
- **매칭 결과 (카드):**
  - **[타이레놀이알서방정]** (해열진통제 / 얀센) - [O] 선택됨
  - **[소론도정]** (부신피질호르몬제 / 유한양행) - [O] 선택됨
  - **[스티렌투엑스정]** (소화성궤양용제 / 동아에스티) - [O] 선택됨
- **정보 요약:** "총 3가지 약에 대한 정보를 찾았습니다."

**사용자 액션:**
- "맞춤 분석 결과 보기" 클릭.

### [Step 4] 최종 복약 지도 리포트 (Medication Report)
**화면 구성:**
- **복용 가이드:** "하루 3번, 식후 30분에 드세요." (일반적 용법)
- **주요 효능:**
  - **타이레놀:** 열을 내리고 통증을 줄여줍니다.
  - **소론도정:** 염증을 가라앉힙니다.
  - **스티렌:** 위 점막을 보호합니다.
- **주의사항 (Food Interaction):**
  > 🚫 **"술은 절대 안 돼요!"** (타이레놀 + 알코올 = 간 손상 위험)
  > ⚠️ **"속이 쓰릴 수 있어요."** (소론도정 부작용)
- **동의보감 기반 식이요법:**
  > "위 보호를 위해 **'양배추'**나 **'마'**를 함께 드시면 속이 한결 편안해질 거예요."

## 4. 공통 요소 (Common UI)
- **로딩 애니메이션:** 지루함을 줄이는 귀여운 캐릭터나 팁 문구 표시.
  ("Tip: 물을 많이 마시면 감기 회복에 좋아요!")
- **저장하기:** "이 분석 결과를 '나의 건강 기록'에 저장할까요?"
- **공유하기:** 카카오톡 등으로 가족에게 공유.
